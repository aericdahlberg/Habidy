## Database Schema

### `users`
```sql
id                  uuid PRIMARY KEY DEFAULT gen_random_uuid()
email               text UNIQUE NOT NULL
name                text
identity_statement  text        -- "I want to be someone who..."
goal_category       text        -- health | career | relationships | creativity | mindset | other
friction_point      text        -- "One thing that doesn't match who I want to be"
time_available      text        -- 5min | 15min | 30min | 60min+
onboarding_done     boolean DEFAULT false
new_user            boolean DEFAULT true   -- set to false after /welcome is viewed
created_at          timestamptz DEFAULT now()
```
> Row level security: users can only read/write their own row.
> `new_user` is checked at login: true → /welcome, false → /dashboard.

### `habits`
```sql
id                  uuid PRIMARY KEY DEFAULT gen_random_uuid()
user_id             uuid REFERENCES users NOT NULL
name                text NOT NULL
identity_label      text        -- "I am a runner" — generated by Architect
identity_link       text        -- "I am someone who takes care of their body" (legacy)
cue                 text        -- "After I make coffee"
craving             text
action              text
reward              text
two_minute_version  text
time_of_day         text        -- morning | afternoon | evening | anytime
goal_category       text
color               text        -- hex color matching category
is_active           boolean DEFAULT true
created_at          timestamptz DEFAULT now()
```

### `habit_logs`
```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
habit_id    uuid REFERENCES habits NOT NULL
user_id     uuid REFERENCES users NOT NULL
date        date NOT NULL
completed   boolean NOT NULL
logged_at   timestamptz DEFAULT now()
```

### `habit_survey_responses`
```sql
id                uuid PRIMARY KEY DEFAULT gen_random_uuid()
user_id           uuid REFERENCES users NOT NULL
habit_id          uuid REFERENCES habits NOT NULL
date              date NOT NULL
went_right        text        -- free text, optional
went_wrong        text        -- free text, optional
completion_level  text        -- 'full' | 'partial' | 'none'
created_at        timestamptz DEFAULT now()
```
> Populated when user swipes UP on a habit card.
> After save, POST to /api/explore so it feeds into user profile context.

### `proposed_habits`
```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
user_id     uuid REFERENCES users NOT NULL
habit_data  jsonb NOT NULL   -- full habit object from Architect
created_at  timestamptz DEFAULT now()
selected    boolean DEFAULT false
```
> Habits Architect suggested but user did not select.
> Surfaced in /add-habit when user hits 7-day streak.

### `conversation_memory`
```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
user_id     uuid REFERENCES users NOT NULL
agent       text NOT NULL   -- 'constellation' | 'architect'
summary     text            -- structured JSON or 2-3 sentence AI summary
created_at  timestamptz DEFAULT now()
```

### `user_reflections`
```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
user_id     uuid REFERENCES users NOT NULL
content     text NOT NULL
created_at  timestamptz DEFAULT now()
```
> Row level security: users can only read/write their own rows.

### `user_profile_context`
```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
user_id     uuid REFERENCES users NOT NULL UNIQUE
summary     text            -- running AI-generated profile summary
updated_at  timestamptz DEFAULT now()
```
> One row per user. Updated after each new reflection via /api/explore.
> Read by all agents via getProfileContext(user_id).

### `tool_logs` (agent guard layer)
```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
user_id     uuid
agent       text
tool_name   text
input       jsonb
output      jsonb
success     boolean
error       text
duration_ms int
created_at  timestamptz DEFAULT now()
```

## API Routes

### Auth (Supabase built-in — no custom routes needed)
Supabase Auth handles signup/login/logout via the client SDK.
Session is stored via Supabase's built-in cookie/token mechanism.
`user.id` from the Supabase session replaces all `user_id` localStorage usage.

### Onboarding
```
POST  /api/onboarding     -- save identity, goal_category, friction_point, time_available
                          -- sets onboarding_done = true on the users row
```

### Habits
```
GET   /api/habits              -- get user's active habits
POST  /api/habits              -- create habit(s) — accepts single object OR array
POST  /api/habits/[id]/log     -- log completion or skip for today
GET   /api/habits/[id]/streak  -- returns current streak + last 7 days
GET   /api/habits/proposed     -- returns proposed_habits for this user (selected = false)
```

### Agents
```
POST  /api/agents/constellation   -- send message, get response (Insights agent)
POST  /api/agents/architect       -- send message, get response
```

### Memory
```
GET   /api/memory/[agent]         -- get latest summary for this agent
POST  /api/memory/[agent]         -- save session summary
```

### Explore (Reflection)
```
POST  /api/explore   -- saves a new user_reflections row,
                     -- then calls Claude to regenerate user_profile_context.summary
GET   /api/explore   -- returns all reflections for this user, reverse-chron
```

### Survey
```
POST  /api/survey    -- saves a habit_survey_responses row,
                     -- then calls POST /api/explore with the survey content
```

## Swipe Mechanic

| Gesture | Action |
|---------|--------|
| Swipe right | Complete — POST /api/habits/[id]/log with completed: true |
| Swipe left  | Skip — POST /api/habits/[id]/log with completed: false |
| Swipe up    | Open quick survey bottom sheet (3 questions), then POST /api/survey |

## Helper Function

```typescript
// lib/supabase.ts
export async function getProfileContext(userId: string): Promise<string> {
  const { data } = await supabase
    .from('user_profile_context')
    .select('summary')
    .eq('user_id', userId)
    .single()
  return data?.summary ?? ''
}
```
