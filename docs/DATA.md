Add tables when you need them.

## Database Schema

### `users`
```sql
id                  uuid PRIMARY KEY DEFAULT gen_random_uuid()
email               text UNIQUE NOT NULL
name                text
identity_statement  text        -- "I want to be someone who..."
goal_category       text        -- Health & Fitness | Career & Learning | etc.
friction_point      text        -- "One thing that doesn't match who I want to be"
time_available      text        -- 5min | 15min | 30min | 60min+
new_user            boolean DEFAULT true   -- set to false after /welcome is viewed
onboarding_done     boolean DEFAULT false
created_at          timestamptz DEFAULT now()
```
> Row level security enabled. Server-side routes use adminClient() to bypass RLS.
> `new_user` is checked at login: true → /welcome, false → /dashboard.

### `habits`
```sql
id                  uuid PRIMARY KEY DEFAULT gen_random_uuid()
user_id             uuid REFERENCES users NOT NULL
habit_name          text NOT NULL           -- display name (not `name`)
identity_label      text                    -- "I am a ___" — generated by Architect
cue                 text                    -- "After I make coffee, I will..."
craving             text
action              text
reward              text
two_minute_version  text
category            text                    -- Health & Fitness | Career & Learning | etc.
color               text                    -- hex, optional
time_of_day         text DEFAULT 'anytime'  -- morning | afternoon | evening | anytime
is_active           boolean DEFAULT true
created_at          timestamptz DEFAULT now()
```
> Column is `habit_name` not `name`. Category column is `category` not `goal_category`.
> Color is stored but category color is applied via CATEGORY_STYLES in HabitCard.tsx.

### `habit_logs`
```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
habit_id    uuid REFERENCES habits NOT NULL
user_id     uuid REFERENCES users NOT NULL
date        text NOT NULL       -- ISO date string "YYYY-MM-DD"
completed   boolean NOT NULL
logged_at   timestamptz DEFAULT now()
```

### `habit_survey_responses`
```sql
id                uuid PRIMARY KEY DEFAULT gen_random_uuid()
user_id           uuid REFERENCES users NOT NULL
habit_id          uuid REFERENCES habits NOT NULL
date              text NOT NULL
went_right        text        -- free text, optional
went_wrong        text        -- free text, optional
completion_level  text        -- 'full' | 'partial' | 'none'
created_at        timestamptz DEFAULT now()
```
> Populated when user swipes UP on a habit card.
> After save, content is also POSTed to /api/explore to feed into user profile context.

### `proposed_habits`
```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
user_id     uuid REFERENCES users NOT NULL
habit_data  jsonb NOT NULL   -- full habit object from Architect
selected    boolean DEFAULT false
created_at  timestamptz DEFAULT now()
```
> Habits Architect suggested but user did not select at that time.
> Surfaced in /add-habit when user hits 7-day streak on any habit.

### `conversation_memory`
```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
user_id     uuid REFERENCES users NOT NULL
agent       text NOT NULL   -- 'identity-gatherer' | 'architect'
summary     text            -- JSON fields + plain-text recap merged into one object
created_at  timestamptz DEFAULT now()
```
> Agent value is `'identity-gatherer'` (not `'constellation'`).

### `user_reflections`
```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
user_id     uuid REFERENCES users NOT NULL
content     text NOT NULL
created_at  timestamptz DEFAULT now()
```

### `user_profile_context`
```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
user_id     uuid REFERENCES users NOT NULL UNIQUE
summary     text            -- running AI-generated profile summary
updated_at  timestamptz DEFAULT now()
```
> One row per user. Updated after each new reflection or survey via /api/explore.
> Read by all agents via getProfileContext(user_id).

### `tool_logs`
```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
user_id     uuid
agent       text            -- agent name or 'session_end' marker
tool_name   text
input       jsonb
output      jsonb
success     boolean
error       text
duration_ms int
created_at  timestamptz DEFAULT now()
```

## API Routes

### Auth (Supabase built-in — no custom routes needed)
Supabase Auth handles signup/login/logout via the client SDK.
Session stored via Supabase's built-in cookie mechanism.
`user.id` from the Supabase session is used everywhere.

### Onboarding
```
POST  /api/onboarding     -- save identity_statement, goal_category, friction_point,
                          -- time_available; upserts users row; sets new_user = false
```

### Habits
```
GET   /api/habits              -- get user's active habits (is_active = true)
POST  /api/habits              -- create habit(s) — accepts single object OR array
POST  /api/habits/[id]/log     -- log completion or skip for today (upserts)
GET   /api/habits/[id]/streak  -- returns current streak + last 7 days
POST  /api/habits/survey       -- save habit_survey_responses row
```

### Agents
```
POST  /api/agents/constellation   -- Identity Gatherer — POST messages, get reply
POST  /api/agents/architect       -- Architect — POST messages, get reply
```

### Memory
```
GET   /api/memory/[agent]         -- get latest summary for this agent + user
POST  /api/memory/[agent]         -- save session summary
```

### Explore (Reflection)
```
POST  /api/explore   -- saves a new user_reflections row,
                     -- then calls Claude to regenerate user_profile_context.summary,
                     -- returns the new summary
```

## Swipe Mechanic

| Gesture | Action |
|---------|--------|
| Swipe right | Complete — POST /api/habits/[id]/log with completed: true |
| Swipe left  | Skip — POST /api/habits/[id]/log with completed: false |
| Swipe up    | Open survey bottom sheet (3 questions), then POST /api/habits/survey |
